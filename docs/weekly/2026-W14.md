## 本周最重要的进展
- 诊断链路完成一轮从解析、排序到默认层文案预算的收口：复杂混合输入更稳定保留技术主问题，quick / standard / deep 主线一致，低证据场景的默认输出更短、更聚焦可执行动作。
- 研究执行闭环补上观测与导出：study flush 失败被分桶为 `network_error`、`http_non_2xx`、`beacon_rejected`，并进入本地导出摘要，便于正式研究前检查日志可靠性。
- 内容池扩充并清洗到 1065 条后，诊断推荐与训练计划候选同步扩池，计划文案也进一步压缩成更像教练口令的可执行提示。

## 功能推进
- 诊断机制先做了混合输入判别、子句覆盖奖励、技术/心理冲突惩罚等迭代，减少误判到兜底心理标签；随后又加入 summary 预算与 `detailedSummary` 分层，降低默认层信息密度。
- 推荐与计划匹配连续增强：推荐列表增加创作者去集中化与场景直链优先，计划评分更依赖诊断上下文词命中，候选池阈值提升并保留 seeded 内容优先，避免“去重后更不贴题”。
- 内容与文档同步推进：内容库从 673 扩到 1065，legacy canonical 缺失与高集中簇告警清零；`[[roadmap/content-expansion-1000]]`、`[[features/youtube-platform-spec]]`、`[[definition-of-done]]` 等文档入口与约束同步更新。

## 关键决策
- 诊断默认层优先保“可执行的主动作”，详细解释按需展开，而不是默认堆叠长文案。
- quick / standard / deep 允许解释深度不同，但不允许 `problemTag` 和 `primaryNextStep` 漂移。
- 观测增强优先做在聚合与导出层，不更改既有 study event schema。
- 扩库后计划候选继续让 seeded 内容优先，优先保证主题贴合，再处理去重复与多样性。

## 暴露的问题 / 风险
- 内容池里仍有历史 search URL 条目，虽然诊断直链过滤已规避，但后续仍需继续替换与审计。
- YouTube 缩略图策略仍停留在 `mqdefault`，尚未落地 `maxresdefault -> hqdefault` fallback。
- 平台分布当前偏向 YouTube；如果要严格回到 600 / 400 目标，还需要后续再做分布回调。
- `scripts/verify-creators-existence.ts` 在历史终端记录里出现过 exit code 1，建议在 CI 或干净终端补一次留档。
- 需要确认诊断推荐扩池与当前 study 约束是否完全一致，避免“仅 `src/data/contents.ts` 直链”与“`contents + expandedContents` 直链”两套口径并存。

## 本周形成的结论
- 本周的真实变化不是单点功能新增，而是把研究模式主链路从“可用”进一步收口到“更稳、更短、更可验证”：诊断方向更稳定，计划更可执行，研究日志更可检查。
- 内容扩库只有和清洗、直链过滤、候选策略、文案压缩一起完成时，才真正提高研究阶段的 routing 质量；单纯“更多视频”不是本周核心价值。
- 下一阶段重点应从继续堆功能，转向研究执行闭环验证，包括双语走查、flush 分桶检查和剩余数据质量审计。

## 下周建议重点
- 按 `[[roadmap/current]]` 的研究执行闭环继续推进：完成中英 walkthrough，检查 flush fallback 分桶分布，只在研究反馈明确时微调 summary 预算或计划文案。
- 继续清理剩余 search URL、补齐 YouTube 缩略图 fallback，并确认诊断推荐数据源约束与 study 文档口径一致。
