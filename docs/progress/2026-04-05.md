## 今日核心进展
- 基本按 `[[roadmap/next-steps]]` 的四个阶段把主工作做完了：先清理低价值噪音，再重做 assessment，再补强 assessment -> plan，最后补齐 diagnosis -> plan 的模板与 rationale。
- `水平评估 / Assessment` 从旧的浅层分支问卷收口成更强的 9 题梯式评估：新增 movement / pressure 粗筛，分支题扩到 4 题，`4.5` 终于可达，`score=2` 不再消失。
- `问题诊断 -> 训练计划` 的连接明显变强：6 个常见 problemTag 补上了专用模板，assessment 弱项会进入 plan context，diagnosis 也会在 `/plan` 上直接说明“为什么是这份计划”。

## 具体变化
### 产品/功能
- 去掉 assessment 里实际没有价值的 gender friction，当前活跃评估流不再被无用 profile 步骤打断。
- assessment 结果页现在会明确显示：
  - 个性化 summary
  - 薄弱项
  - 待观察项
  - 四档能力状态（`强项 / 正常 / 待提升 / 薄弱`）
- `/plan` 不再只是展示模板标题；assessment 来源会显示弱项导向，diagnosis 来源会显示与当前诊断、主动作、deep context 对应的 rationale。
- 补齐 6 个原来会掉回 generic fallback 的诊断计划：
  - `rally-consistency`
  - `forehand-no-power`
  - `balls-too-short`
  - `return-under-pressure`
  - `cant-hit-lob`
  - `stamina-drop`

### 技术实现
- 清理了 root / snapshot 噪音：
  - 根目录 planning docs 归档到 `docs/roadmap/archive/`
  - 删除旧 study snapshot creator / metadata
  - `.env.example` 对齐到 `2026-03-31-v1`
  - 修正 snapshot registry 一致性
- assessment engine 重写重点：
  - coarse 题扩到 5 题，fine 分支扩到每支 4 题
  - Branch C 阈值重平衡，`4.5` 不再是 dead code
  - 重复维度不再把低信号平均抹平
  - personalized summary 改为直接引用最弱两项
- assessment UI 收口重点：
  - 去掉 gender 专用布局
  - draft 恢复逻辑兼容旧 flow，同时不再误伤当前 draft step
  - 结果页与 breakdown 全部跟随当前语言渲染，避免 mixed-language output
- assessment -> plan 衔接重点：
  - `PlanContext` 新增 `weakDimensions` / `observationDimensions` / `rationale`
  - assessment 结果页生成 `/plan` 链接时带上更强的 assessment context
  - plan draft 持久化已覆盖 richer context
- diagnosis -> plan 收口重点：
  - diagnosis-origin rationale 正式进入 shared plan summary contract
  - `DiagnoseResult` 在有 deep context 时会优先用更深的 source input 推导 plan context
  - assessment path 与 diagnosis path 的 summary contract 都补了回归测试
- 今日这批实现主要靠 targeted vitest + 多轮 `npm run build` 收口；`npm run lint` 依旧不适合作为可靠入口，因为 repo 还会进入 Next.js ESLint 初始化交互。

## 与昨天相比的真实变化
- 昨天（Apr 4）主要是在 deep diagnosis / scenario reconstruction 上把 `/diagnose` 和 `/plan` 的深层 handoff 做通；今天的真实推进是把 `next-steps` 里指出的“结构性缺口”逐项补齐，并真正落到 `main`。
- 昨天的重点是 serve gold slice 深化；今天的重点是把评估、普通 diagnosis、assessment-origin plan、diagnosis-origin plan 这几条主链一起收口，不再只让 deep serve 路径质量高。

  - assessment 不再太浅
  - 常见 diagnosis 不再掉到 generic plan
  - plan 不再只“看起来像有关”，而是会解释为什么这样排
