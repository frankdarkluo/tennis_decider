# TennisLevel — UI 大幅精简任务包

> 核心原则：大道至简。删比加重要。
> 真实反馈来源：40 岁男性球友现场测试。
> 核心问题：文字太多看不下去；评估题选项太细后面瞎选；博主信息太满不想细看。
> 每个改动都围绕一个问题：用户愿不愿意继续往下走？

---

## 第一块：水平评估大改（最高优先级）

真实反馈："头两道题还认真做，后面几题就开始瞎选了。"

### 任务 U1：评估流程重新设计

**当前状态：** 8 道题，每题 4-5 个长文字选项。
**目标状态：** 2 道画像题 + 4 道核心题，每题最多 3 个短选项。

#### 第一部分：前置 2 道画像题（不计入评分）

这两道题的目的是破冰，让用户觉得"这个很轻"。

**题 0a：性别**

```
你的性别？

[ 男 ]  [ 女 ]
```

- 两个大按钮并排，点击即跳下一题
- 不需要"下一题"按钮
- 按钮尺寸：至少 120px 宽，44px 高

**题 0b：打球年限**

```
打了多久网球？

[========●==========]
      2 年

0   半年  1年  2年  5年  10年  20+
```

- 用 range slider，不用输入框
- 为什么 slider 而不是输入框：用户不需要精确数字，滑一下比打字快 3 秒
- 为什么不用下拉框：下拉框在手机上需要弹出 picker，体验重
- 默认值：2 年
- 刻度标记用灰色小字
- 滑动后自动跳下一题（延迟 500ms）

#### 第二部分：核心评估题从 8 题砍到 4 题

只保留最能区分 2.5 / 3.0 / 3.5 / 4.0 的 4 道题。

**新题 1：正反手稳定性**

```
你和朋友对打时，通常能连续打多少拍？

[ 5 拍以内 ]    [ 5-15 拍 ]    [ 15 拍以上 ]
```

**新题 2：发球**

```
你的发球大概是什么状态？

[ 经常发不进 ]    [ 能发进但没变化 ]    [ 有旋转和落点控制 ]
```

**新题 3：移动和脚步**

```
打球时你的脚步情况？

[ 经常来不及 ]    [ 能到但不舒服 ]    [ 基本能提前到位 ]
```

**新题 4：比赛意识**

```
你在比赛或对打中的状态？

[ 只想着把球打过去 ]    [ 会想打方向但不稳定 ]    [ 能有意识组织一分 ]
```

#### 选项设计规则

- 每题只有 3 个选项（不是 4-5 个）
- 每个选项最多 10 个中文字
- 选项用卡片式按钮（不是 radio button）
- 选中后自动跳下一题（延迟 300ms，有选中动画）
- 不需要"下一题"按钮

#### 进度指示

- 不显示"第 X 题 / 共 Y 题"
- 改用一条细进度条在顶部，无数字
- 做到第 3 题时底部浮现淡灰色文字："快完成了"

#### 评分逻辑调整

修改 `src/lib/assessment.ts`：

```typescript
// 新的评分映射
// 每题 3 个选项，分别对应 1 / 2 / 3 分
// 总分 4-12 分
// 4-5 分 → 2.5
// 6-7 分 → 3.0
// 8-9 分 → 3.5
// 10-12 分 → 4.0
```

画像题（性别、年限）不计入评分，但保存到评估结果中供后续分析。

#### 需要改动的文件

- `src/data/assessmentQuestions.ts` → 全部重写为 4 道新题
- `src/app/assessment/page.tsx` → 前置 2 道画像题 + 自动跳转逻辑
- `src/components/assessment/QuestionCard.tsx` → 改为 3 选项卡片式
- `src/components/assessment/AssessmentProgress.tsx` → 改为纯进度条无数字
- `src/lib/assessment.ts` → 新的 4 题评分算法

#### 验收标准

- 总共 6 步（2 画像 + 4 评估），30 秒内可完成
- 每个选项 ≤ 10 个中文字
- 无"下一题"按钮，选中自动跳转
- 进度条无数字
- 评分输出与之前格式兼容（level + 维度分数）

```typescript
function filterContents(contents: Content[], query: string): Content[] {
  if (!query.trim()) return contents;
  const q = query.toLowerCase();
  return contents.filter(c =>
    c.title.toLowerCase().includes(q) ||
    c.skills?.some(s => s.toLowerCase().includes(q)) ||
    c.useCases?.some(u => u.toLowerCase().includes(q)) ||
    (c.coachReason || '').toLowerCase().includes(q) ||
    getCreatorName(c.creatorId).toLowerCase().includes(q)
  );
}
```

#### 实现方式

```tsx
const [layer, setLayer] = useState<1 | 2 | 3>(1);

return (
  <>
    {/* 第一层：永远显示 */}
    <div>
      <h2>你现在最该先改的是：</h2>

      </div>
    )}

    {layer === 2 && (

    )}

    {/* 第三层 */}
    {layer >= 3 && (
      <div>
```

#### 全局规则

- 段落 → 句子
- 句子 → 短句
- 能用 10 个字说清楚的不要用 20 个字
- 删掉所有"解释系统在做什么"的文字，只保留"帮用户做下一步"的文字

**这轮改动的核心不是加东西，而是删东西。每个页面都要过一遍"用户第一眼看到什么"的测试：如果第一眼看到的不是他该做的下一步，那页面上的其他东西就应该被删掉或折叠。**
