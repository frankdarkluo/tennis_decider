# TennisLevel 项目进度总结

更新时间：2026-03-24

## 1. 当前项目是什么

TennisLevel 是一个面向业余网球用户的训练决策型 MVP。它不是单纯的内容库，而是把用户的真实困惑逐步转成下一步训练建议：

1. 先做 1 分钟水平评估
2. 再用一句话描述问题
3. 系统给出问题诊断和推荐内容
4. 最后生成 7 天训练计划

当前版本已经具备完整闭环：

- 首页承接用户注意力并分发入口
- 评估页输出参考等级和短板
- 诊断页输出问题归因、修正点和推荐内容
- 内容库与博主榜支持筛选浏览
- 训练计划页能根据问题和等级生成方案
- 登录后可保存评估、诊断历史、收藏和训练计划

## 4. 当前代码逻辑怎么运转

### 页面层

- `/`
  首页，承接用户注意力，分发到评估 / 诊断 / 内容 / 博主
- `/assessment`
  问卷评估页，计算参考等级和维度分数
- `/assessment/result`
  展示评估结果，登录后优先读取云端最近记录
- `/diagnose`
  根据用户一句话输入做规则匹配与推荐
- `/library`
  内容库，支持按等级 / 技术 / 平台 / 语言 / 类型 / 创作者筛选
- `/rankings`
  博主榜，支持等级、技术方向、风格等筛选
- `/plan`
  根据评估或诊断上下文生成 7 天训练计划

### 核心业务逻辑

- `src/lib/assessment.ts`
  负责评估分数计算、等级映射、维度总结
- `src/lib/diagnosis.ts`
  负责输入标准化、规则评分、同义匹配、等级优先推荐、兜底推荐
- `src/lib/plans.ts`
  负责根据问题标签和等级生成训练计划
- `src/lib/userData.ts`
  负责和 Supabase 的用户数据读写

### 数据层

- `src/data/assessmentQuestions.ts`
- `src/data/diagnosisRules.ts`
- `src/data/contents.ts`
- `src/data/creators.ts`
- `src/data/planTemplates.ts`

当前依然是“结构化本地数据 + Supabase 用户数据”的架构，不是后台 CMS 驱动。

## 5. 当前真实进度评价

如果按 0 到 100 分粗略评估：

- 产品闭环：85/100
- 首页与体验：80/100
- 内容可信度：70/100
- 登录价值：75/100
- 工程化能力：75/100

### 为什么已经明显高于早期 MVP

- 产品主流程已经能完整走通
- 首页不再只是展示页，而是有效导流页
- 内容推荐不再是纯占位
- 登录不再只是“能登录”，而是已经绑定保存价值
- 已经有基础测试和数据校验，不再完全靠手工检查

### 还没到更高分的原因

- 数据量还不够厚，尤其是内容条目和计划模板数量偏少
- 推荐逻辑仍然是规则型 + 数据映射型，不是学习型系统
- 还没有“我的主页”这样的统一个人中心
- 没有后台管理内容的能力
- 真实用户反馈数据还比较少

---

## 6. 当前最大的短板

当前最大的短板已经不是首页，而是下面三件事：

1. 内容规模还不够大
2. 个性化留存能力还不够强
3. 真实用户反馈还不够多

更具体一点说：

- 内容是可信的，但还不够丰富
- 登录已经有价值，但“我的记录”还没有前台统一承接页
- 工程基础已起步，但还没有埋点、监控、后台

---

## 7. 下一阶段最建议优先做什么

### 第一优先级：继续补内容密度

建议优先补这三类：

- 高频问题对应的中文内容
- 4.0 / 4.5 球员也能看得上的进阶内容
- 更多训练计划模板

目标不是“无限堆内容”，而是继续围绕规则和问题标签，把每个高频问题的内容密度补厚。

### 第二优先级：做“我的主页”

现在保存能力已经有了，下一步最值得补的是前台承接：

- 最近一次评估结果
- 最近诊断记录
- 收藏内容
- 已保存训练计划

这会让登录从“有功能”升级成“有持续使用理由”。

### 第三优先级：拿真实用户试跑

建议找 5 到 10 个球友，完整走一遍：

- 首页
- 评估
- 诊断
- 推荐内容
- 训练计划

观察他们卡在哪里，觉得哪里“像教练”，哪里“还不够准”。

### 第四优先级：再做更深的工程化

包括：

- 埋点
- 错误监控
- 部署流程
- 内容维护后台

## 9. 风险与注意点

- Supabase 数据库能力依赖手动执行 SQL 初始化
- 内容和计划模板仍然主要靠代码手工维护
- 推荐逻辑虽然可用，但还依赖规则质量和内容密度
- 如果后面继续快速加功能，需要尽早补个人中心和后台数据维护能力

---

## 10. 一句话总结

**TennisLevel 已经从“概念页 + 假数据”阶段，进入到“可演示、可试用、可继续打磨成真实产品”的阶段。下一步最值得投入的，是继续补内容密度、做个人中心，并尽快拿真实用户反馈。**
